# Support

This document explains how to get help with Rumba.

## GitHub Discussions

For questions, ideas, and general conversation, use
[GitHub Discussions](https://github.com/orgs/rumba-id/discussions).

Categories:

- **Announcements** - Project news and updates
- **Q&A** - Technical questions
- **Ideas** - Feature suggestions and feedback
- **Show and Tell** - Share your Rumba deployments

Before posting:

1. Search existing discussions
2. Check the documentation
3. If your question is new, post it

## GitHub Issues

Use issues for:

- **Bug reports** - Something is broken
- **Feature requests** - Specific functionality you need

Each repository has issue templates to guide your report.

### Bug Reports

Include:

- Expected vs actual behavior
- Steps to reproduce
- Environment details (OS, Rumba version, etc.)
- Relevant logs or error messages

### Feature Requests

Include:

- What you want to accomplish
- Why existing functionality does not work
- How you envision the feature working

## Response Times

The team is small and development is ongoing. Response times vary:

- Security issues are prioritized
- Bug reports with clear reproduction steps get faster responses
- Feature requests are tracked but may take time to evaluate

## What We Cannot Help With

- General Rust or PostgreSQL questions (use Stack Overflow)
- Issues in third-party integrations
- Deployment problems specific to your infrastructure
